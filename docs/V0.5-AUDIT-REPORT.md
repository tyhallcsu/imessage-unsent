## Documented-only (open, prioritized)

Highest first. None is a High; none blocks v0.5.0.

1. **D-12 — BoundedProcessRunner grandchild kill + pid-reuse TOCTOU.** SIGTERM/
   SIGKILL hit only the direct child; orphaned `wal_extract.py` grandchildren
   can burn CPU unbounded, and the unguarded kill races pid reuse (remote).
   *Priority raised this cycle:* the missing process-group kill is also the
   root cause of a pre-existing ~1-in-3-under-load flake in
   `BoundedProcessRunnerTests.testHangingChildTimesOutIsKilledAndRunnerStaysUsable`
   (the orphaned `sleep` races the `done` marker). Fix via
   `POSIX_SPAWN_SETSID` + `killpg`; makes that test deterministic.
2. **D-7 — daemon log writes `handle=`/`guid=` cleartext** to `~/Library/Logs`
   with no rotation. Redact at info level (hash), rotate by size.
3. **D-13 — no detection sweep at startup:** retractions that occurred while
   the daemon was down wait for the next WAL write. One `handleWalChange`
   call at the end of `startWalWatcher` fixes it.
4. **G-12 — fire-and-forget `Task.sleep` status resets** race on rapid
   re-trigger (cosmetic truncation of transient labels). Generation token.
5. **G-13 — archive-stats disk I/O in `HistoryWindow` body** on first render /
   post-invalidation (millisecond-scale, bounded at 50 archives).
6. **R-9 — release.yml residue:** job outputs interpolated into `run:` (safe
   only while the semver guard stays anchored — move to env like INPUT_TAG),
   workflow-scoped `contents: write`, release binaries from restored caches,
   broad `sed s/VERSION/` in notes post-processing.
7. **R-10 — README link checker skips HTML `<img src=…>`** — the three hero
   images are unchecked.
8. **D-10 — perms residuals:** pre-#129 installs keep 0755 `~/.config/…` dirs;
   quarantined corrupt state files keep their old mode.
9. **R-14 — `edit-history.py` defaults to the live chat.db** (read-only URI,
   row-level data only — WAL not involved) — divergence from the
   snapshot-first convention; snapshot by default or document.
10. **R-17 — recover.sh conflates sqlite errors with "no candidates"**
    (`not_in_local_wal`, exit 0) — add a `snapshot_unreadable` category.
11. **R-15 — sign-release keychain-list rebuild** breaks on multi-keychain
    dev Macs (command-substitution collapses the list).
12. **G-11 remainder — accessibility identifiers + VoiceOver announcements**
    for transient confirmations; enables XCUITest targeting.
13. **G-18 — `.searchable` on a plain Window** may not render on macOS 13
    (speculative; verify on the min target).
14. **G-20 — presentation polish:** raw ISO-8601 timestamps in Settings/detail,
    hand-rolled MB formatting in one spot, no localization scaffolding.
15. **D-14 — installer nits:** plist templating does no XML escaping of $HOME;
    `cp -R scripts/.` ships installers/`__pycache__` into the install tree
    (F-L13).
16. **INFO:** control-socket 1-byte reads allow a same-uid slow-loris on the 4
    worker slots (2 s idle timeout bounds it); lexicographic archive sort
    mis-orders same-second rowids 9/10 at retention boundaries; `KeepAlive=true`
    crash-loops on startup-throwing config errors at launchd's 10 s throttle;
    `defaultRecoverScriptURL` probes CWD first (unexploitable under launchd).

## Deliberate scope cuts (recorded decisions)

- `statusInfo.lastError` does **not** trip the menu attention state: the daemon
  never clears `lastError` after recovery, so it would permanently alarm the
  icon after any transient error. Needs a daemon-side clear-on-success first.
- Single archive delete/compact remain synchronous (bounded 1 s user actions);
  only the poll loop and batch compaction moved off the main actor (#153).
- The unreachable `.detecting`/`.idle` daemon states (ControlServer hardcodes
  `state:"watching"`) stay in the GUI vocabulary for a future daemon that
  reports real states; they display nothing false today.
